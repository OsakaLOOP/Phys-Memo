一个基于关系型数据库存储的分块式知识推导管理和著作标识模块. 支持即时 api 操作和后端异步 Workers 接入(如排名计算, 冗余清洗).

用于前端一个概念展示的所有md-katex域将被统一按块切分, 存储为 atom. 对多行混排的域, 即公式栏和笔记栏, 按照markdown的空行段落和多行公式的双$$划分. 对标签式展示,且仅单行的域, 比如限定条件栏,关系栏和引用栏, 我们统一确保按照标签分条单行输入, 每行对应的原始纯文本或者关系栏的原始json格式被分别存储到对应类别的Atom. 这一部分, 前端的界面和部分展示方式,数据结构需要迁移.

用户可以选择保存或者发布, 均分块保存并打包为atom, 只是后者对全社区可见. 编辑器中的更改和历史将仅在工作区保留. 概念根会记忆其所有分支的最新atom, 如果加入新的atom就更新列表, 这样维持虚拟的branch结构, 不用增加负担. 一个概念根下, 多个分支可以共存, 或者中途创建属于自己分支, 用户也可以对比其他分支并cherry-pick, 记录在attr字段, 但必须是整块摘取然后编辑, attr根据来源和新的编辑自动计算更新. 用户可以在社区为任何atom vote up/down, 分数由worker定时累加, 优先展示分数最高的. 用户也可以在不创建新branch的前提下, 为原主人提供patch并更新版权归属.
