## 2025-04-06 - [Exponential Recursion in Stringify]
**Learning:** Found a major performance bottleneck specific to this codebase's architecture in `deterministicStringify` within `src/attrstrand/utils.ts`. The original implementation used recursive `JSON.parse(deterministicStringify(...))` calls on every level of nested objects. This forced redundant serialization and parsing at every depth, leading to $O(N^D)$ exponential overhead (where D is depth), severely degrading performance for large nested data structures common in this application's state.
**Action:** Always sort objects purely in memory using a simple recursive mapping function (`sortObject`), and only call `JSON.stringify` once at the very end. This reduces the time complexity to $O(N \log N)$ (due to key sorting) and avoids redundant string allocations.
